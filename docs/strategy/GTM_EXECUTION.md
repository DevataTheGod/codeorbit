# GTM Execution

The actual path to first customers.

---

## Primary Channel: PSIT Alumni Network

**Why This Works**:
- Direct access to bootcamp mentors/instructors
- Personal credibility (seen the AI copy-paste problem)
- Developer community trust
- Warm intro capacity

---

## GTM Sequence

### Week 1-2: Discovery

```
LinkedIn search: "PSIT" + "Bootcamp" + "Mentor" / "Instructor"
├─ Create list of 30-50 people
├─ Note: Name, bootcamp, role, location
├─ Research bootcamp: Size, funding, placement rate
└─ Goal: 5-8 mentor conversations
```

### Week 2-3: Validation

```
Call mentors, ask: "How do you verify understanding?"
├─ Listen for pain (AI, mentor overload)
├─ If pain is high: "Can I talk to your founder?"
├─ Mentor makes warm intro to founder/academic head
└─ Goal: 2-3 founder warm intros
```

### Week 3-4: Closing

```
Pitch founder: "Your mentors are struggling. We help them scale."
├─ Show MVP prototype or video demo
├─ Offer pilot: ₹2.5-5L for 4-8 weeks
└─ Goal: 1 pilot signed
```

### Month 2+: Scale with Proof

```
Pilot runs successfully
├─ Document results (mentor time savings, score accuracy)
├─ Create case study + testimonial
├─ Approach bootcamps 2-5 with proof
└─ Goal: 3-5 paid pilots, ₹15-25L revenue
```

## Call Structure (15-20 minutes)

### Opening (2 min)
"Hi [Name], thanks for making time. I'm building a system to help bootcamp mentors verify student understanding. I know you work at [Bootcamp] as [Role], and I imagine you're dealing with the AI-generated code problem. That's what I want to solve. Do you have 15 minutes?"

### Problem Discovery (5 min)
"Tell me about your biggest challenge with students right now."
- Listen for: Understanding verification, mentor overload, AI issues

"How do you currently know if a student truly understands their code?"
- Listen for: "We don't really know", "We assume"

### Solution Presentation (5 min)
"We're building a system that:
1. Measures student understanding through behavioral telemetry
2. Provides AI mentorship that forces thinking (not copying)
3. Gives mentors a dashboard to see exactly where students struggle"

### Close (3 min)
"Would you be open to a pilot? We're working with [bootcamp name] and seeing [results]. The pilot is ₹2.5-5L for 4-8 weeks with 30-50 students."

## Decision Trees

### If Pilot Succeeds
- Document results
- Create case study
- Approach 5-10 bootcamps with proof
- Scale sales

### If Pilot Partially Succeeds
- Identify what worked, what didn't
- Iterate on product
- Run second pilot with improvements
- Then scale

### If Pilot Fails
- Document what failed
- Understand why
- Decide: iterate or pivot
- Run second pilot if fixable

### If No Pilot Closes in Month 1
- Activate secondary channels
- LinkedIn content marketing
- Developer community outreach
- Cold email to 100+ bootcamps
