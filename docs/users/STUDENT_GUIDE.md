# Student Guide

Welcome to CodeOrbit — your AI-powered learning platform for software engineering.

---

## Getting Started

### 1. Create Your Account

1. Go to [codeorbit.app](https://codeorbit.app)
2. Click **Sign Up**
3. Enter your email and password, or sign up with Google
4. You'll be assigned the **Student** role automatically

### 2. Submit Your Project

After signing up, you'll be redirected to submit your first project:

1. **Project Title** — Give your project a clear name
2. **Project Description** — Explain what you want to build
3. **Tech Stack** — Select your primary technologies
4. **Skill Level** — Beginner, Intermediate, or Advanced
5. **Timeline** — When do you want to complete this?

Click **Submit** and your project will be reviewed by a mentor.

---

## Using the IDE

Once your project is approved, you'll have access to the browser-based IDE.

### File Explorer

- Browse your project files in the sidebar
- Click any file to open it in the editor
- Create new files and folders using the File Operations panel

### Code Editor

- Write and edit code directly in the browser
- Syntax highlighting for TypeScript, JavaScript, and more
- Auto-save to local storage

### Terminal

- Run commands like `ls`, `cd`, `cat`, `touch`, `mkdir`
- View your project structure
- Execute code files

### AI Chat (Orbit)

- Ask Orbit for help with your project
- Orbit will guide you without writing code for you
- Use `@` to mention specific files

## Understanding Your Score

CodeOrbit calculates an **Understanding Score** based on:

| Factor | Weight | What It Measures |
|--------|--------|------------------|
| Engagement | 30% | Orbit interactions, typing time, task attempts |
| Reflection Quality | 40% | How well you explain your code |
| Progress | 30% | Milestones completed |

### Risk Levels

| Level | Score Range | Meaning |
|-------|-------------|---------|
| Mastery | 90-100 | Excellent understanding |
| On Track | 75-89 | Good progress |
| At Risk | 60-74 | Needs attention |
| Struggling | 40-59 | Significant gaps |
| Critical | 0-39 | Immediate support needed |

## Reflection Challenges

When you paste code or complete milestones, Orbit may ask you to explain your work.

### How to Answer Well

1. **Explain the "why"** — Not just what the code does, but why you chose this approach
2. **Mention tradeoffs** — What alternatives did you consider?
3. **Use code references** — Show you understand the implementation
4. **Be specific** — Avoid vague answers

### Good Response Example

> I used `useState` here because I need to track the form input locally. I considered `useReducer` but since there's only one state variable, `useState` is simpler. The `useEffect` hook watches for changes in the input and debounces the API call to avoid excessive requests.

### Poor Response Example

> It handles the form.

## FAQ

### Why is my score low?

Your score is based on reflection quality, engagement, and progress. If you're pasting code without explaining it, your score will be lower.

### Can I see my score history?

Yes. Your mentor can view your score timeline and understand how you're improving over time.

### How do I improve my score?

1. Complete milestones on time
2. Answer reflection challenges thoughtfully
3. Interact with Orbit for guidance
4. Avoid copy-pasting without understanding
